# Glossary — Accessibility Knowledge Base

Source tags: **[P]** = AccessAbility Playbook (ESDC) · **[GD]** = Giving a Damn (Byrne-Haber) · **[Appt]** = Appt Handbook.

**AAC (Augmentative and Alternative Communication)** [P] — non-speech communication: gestures, pictures, letter boards, devices (esdc-ch02).

**Ableism** [GD] — beliefs/practices that devalue and discriminate against people with disabilities; assumes they need to be "fixed" (bh-ch01).

**Ableism substitution test** [GD] — swap "disability" for another group; if the statement becomes unacceptable, it's ableist (bh-ch01).

**Accessibility** [P][Appt] — ability to access without barriers; not a toggle — woven into design/dev/testing (esdc-ch01, appt-ch01).

**Accessibility debt** [GD] — compounding cost/risk of staying inaccessible (bh-ch01).

**Accessibility flywheel** [GD] — self-reinforcing loop: compliance → customers → focus → features → practices (bh-ch02).

**Accessible authentication** [Appt] — allow password managers/copy-paste; don't force memory tests (WCAG 3.3.8, appt-ch05).

**ADA (Americans with Disabilities Act)** [GD][Appt] — US civil-rights law; covers non-federal entities; refers to WCAG (bh-ch01, appt-ch02).

**Alt text** [P][Appt] — text describing meaning of non-text content for screen readers; skip decorative (WCAG 1.1.1; esdc-ch05, appt-ch03).

**Appt-EM** [Appt] — app-specific accessibility evaluation methodology (vs web-only WCAG-EM) (appt-ch07).

**Assistive technology (AT)** [P][Appt] — tools enabling access: screen reader, voice/switch control, keyboard, magnifier (esdc-ch01, appt-ch01).

**Barrier** [P] — obstacle preventing a person with a disability from doing what others take for granted (esdc-ch01).

**Billy's friends** [P] — six personas: Mark, Nora, Luca, Jeremy, Léo, Sophie (esdc-ch02).

**Contrast ratio** [Appt] — text 4.5:1; large/bold text & non-text/graphical 3:1 (WCAG 1.4.3/1.4.11; appt-ch03).

**Congenital vs acquired disability** [GD] — different lived perspectives; ask several people (bh-ch04).

**CRPD** [Appt] — UN Convention on the Rights of Persons with Disabilities; most countries signed (appt-ch02).

**Curb-cut effect** [P][Appt] — an accessibility feature helping more people than intended (esdc-ch04, appt-ch01).

**Dark mode** [Appt] — dark background option; ~30% prefer; helps light-sensitivity/concussion (appt-ch01).

**Design system** [GD] — reusable component library ("software Legos"); one defect scales everywhere (bh-ch04).

**EN 301 549** [Appt] — European accessibility standard with a software/app chapter; maps to WCAG 2.1 criteria (appt-ch02).

**EAA / WAD** [Appt] — European Accessibility Act (business) / Web Accessibility Directive (government) (appt-ch02).

**Episodic disability** [P] — symptoms that come and go (esdc-ch01).

**Focus order** [Appt] — AT navigation order; left→right, top→bottom (WCAG 2.4.3; appt-ch04/ch07).

**Keyboard trap** [Appt] — being unable to escape a component without mouse/touch (WCAG 2.1.2; appt-ch04).

**Maturity Model (W3C)** [Appt] — 7 org dimensions: Communications, Knowledge & Skills, Support, Dev Process, Personnel, Procurement, Culture (appt-ch07).

**Name, Role, Value (NRV)** [Appt] — semantic triple every interactive element must expose to AT (WCAG 4.1.2; appt-ch06).

**Overlay / plugin / widget** [GD] — "one line of code" accessibility vendors; claims "pants on fire"; invite lawsuits (bh-ch01).

**PDF/UA** [GD] — PDF accessibility standard; litmus-test acronym for vendor competence (bh-ch03).

**Person-first language** [P] — "person who is blind," not "the blind" (esdc-ch06).

**Plain language** [P] — clear, concise writing at grade 6–8; not over-simplified (esdc-ch05).

**Release gate** [GD] — go/no-go check ensuring only accessible software ships (bh-ch04).

**Section 508** [Appt] — US law requiring federal government apps to be accessible (appt-ch02).

**Shift Left** [Appt] — fix issues in design, where it's cheapest (appt-ch07).

**Situational disability** [Appt] — environment-induced limitation (can't use sound in a quiet zone) (appt-ch01).

**Status message** [Appt] — dynamic info (error/offline) announced to AT without moving focus (WCAG 4.1.3; appt-ch06).

**Switch control** [Appt] — operate apps via internal (blink/camera) or external (button/joystick) switches (appt-ch01).

**Target size** [Appt] — minimum 24×24 px for activatable elements (WCAG 2.5.8; appt-ch04).

**Text scaling** [Appt] — support ≥200% without truncation/overlap; wrap screens in ScrollView (appt-ch01/ch03).

**TTY / TDD** [P] — Text Telephone for Deaf/hard-of-hearing (esdc-ch02/ch06).

**Universal Design** [P] — designing without barriers; 7 principles; good design for all (esdc-ch04).

**VPAT** [GD] — Voluntary Product Accessibility Template; core accessibility-program artifact (bh-ch03).

**VRS / VRI** [P] — Video Relay Service / Video Remote Interpreting for sign-language users (esdc-ch02/ch06).

**WCAG** [P][GD][Appt] — Web Content Accessibility Guidelines; 2.2 = 4 principles / 13 guidelines / 86 criteria (A/AA/AAA); non-prescriptive; a floor not a ceiling (all books).

**Zero-sum (not)** [GD] — never help one disability group by harming another (bh-ch02).
